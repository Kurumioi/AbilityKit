## 模块解决的问题

移动系统模块把输入移动、技能位移、击退、路径跟随和控制效果抽象为统一的 `IMotionSource`，再由 `MotionPipeline` 根据分组、优先级和叠加策略组合，最后交给 `IMotionSolver` 处理碰撞与约束。

这种设计可以避免移动逻辑分散在多个 MonoBehaviour、技能脚本或碰撞组件中，便于测试、回放、预测和服务器验证。

## 核心承诺

1. **组合**：多个移动来源可以按规则叠加或互斥。
2. **分离**：移动来源、组合策略和碰撞求解相互解耦。
3. **确定性**：输入数组做防御性拷贝，source 支持快照恢复。
4. **低分配**：高频对象接入 `AbilityKit.Core.Pooling` 对象池。

## 当前核心类型

| 类型 | 说明 |
| --- | --- |
| `MotionPipeline` | 移动协调器。 |
| `MotionPipelinePool` | pipeline 与内部集合对象池。 |
| `IMotionSource` | 移动来源接口。 |
| `IMotionSnapshotSource` | source 快照导出/导入接口。 |
| `ConfigurableMotionSolver` | 可配置碰撞/约束求解器。 |
| `IMotionCollisionWorld` | 项目侧碰撞世界适配接口。 |
