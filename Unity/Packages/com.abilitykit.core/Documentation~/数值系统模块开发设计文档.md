## 一、设计理念：为什么要做数值系统？

### 1.1 数值系统 vs 属性系统

```
┌─────────────────────────────────────────────────────────────┐
│                    数值系统 vs 属性系统                       │
│                                                             │
│   这两个系统是互补关系，不是互斥的！                         │
│                                                             │
│   ┌─────────────────────┬─────────────────────────────┐   │
│   │                     │         数值系统              │   │
│   ├─────────────────────┼─────────────────────────────┤   │
│   │ 用途                │ 临时计算、效果加成           │   │
│   │ 生命周期            │ 随用随创建、用完即弃         │   │
│   │ 依赖管理            │ 无                          │   │
│   │ 约束系统            │ 无                          │   │
│   │ 复杂度              │ 轻量、简单                  │   │
│   │ 典型场景            │ 伤害计算、技能加成           │   │
│   └─────────────────────┴─────────────────────────────┘   │
│                                                             │
│   ┌─────────────────────┬─────────────────────────────┐   │
│   │                     │         属性系统              │   │
│   ├─────────────────────┼─────────────────────────────┤   │
│   │ 用途                │ 持久属性、角色数值           │   │
│   │ 生命周期            │ 实体生命周期内常驻           │   │
│   │ 依赖管理            │ 支持（MaxHp → Hp）         │   │
│   │ 约束系统            │ 支持（范围限制）            │   │
│   │ 复杂度              │ 完整、功能丰富              │   │
│   │ 典型场景            │ 角色HP、攻击力、防御力       │   │
│   └─────────────────────┴─────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 传统做法的问题

```
❌ 传统伤害计算的问题：

1. 数值硬编码
   ┌─────────────────────────────────────────────────────────┐
   │ // 伤害计算到处写：                                   │
   │ float CalcDamage(float baseDmg, float bonus) {        │
   │     return (baseDmg + bonus) * damageMultiplier;      │
   │ }                                                     │
   │                                                          │
   │ // 同样的公式到处复制粘贴                            │
   │ // 改一个地方要改N处                                  │
   └─────────────────────────────────────────────────────────┘

2. 加成叠加混乱
   ┌─────────────────────────────────────────────────────────┐
   │ // 伤害加成有多种：                                   │
   │ float totalBonus = baseDamage;                       │
   │ foreach (buff in buffs) {                            │
   │     if (buff.type == AddBonus) totalBonus += buff.val;│  │
   │     if (buff.type == MulBonus) multiplier *= (1+buff.val);│
   │ }                                                     │
   │ finalDmg = totalBonus * multiplier;                   │
   │                                                          │
   │ // 顺序搞错结果就错                                   │
   │ // 乘法加成和加法加成混在一起                         │
   └─────────────────────────────────────────────────────────┘

3. 临时变量满天飞
   ┌─────────────────────────────────────────────────────────┐
   │ // 算个伤害要一堆中间变量：                           │
   │ var base = 100f;                                     │
   │ var flatBonus = GetAllFlatBonus();                   │
   │ var pctBonus = GetAllPctBonus();                     │
   │ var finalBonus = GetAllFinalBonus();                 │
   │ var dmg = (base + flatBonus) * (1 + pctBonus) + finalBonus;│
   │ // 用完就扔，下次再算                                  │
   └─────────────────────────────────────────────────────────┘
```

```

│                                                             │

│   │                                                       │  │

│   │   │                                           │    │  │
│   │   │   NumberValue                             │    │  │
│   │   │                                           │    │  │
│   │   │   Base: 100                              │    │  │

│   │                                                       │  │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解数值系统

### 2.1 什么是"数值（NumberValue）"？

**通俗解释**：NumberValue 是"带加成的计算器"，可以在基础值上叠加多种修饰器。

```
┌─────────────────────────────────────────────────────────────┐
│                    NumberValue = 带加成的计算器              │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  new NumberValue(NumberValueMode.BaseAddMul, 100)   │  │
│   │                                                       │  │
│   │  ┌─────────────────────────────────────────────┐   │  │
│   │  │                                               │   │  │
│   │  │   基础值 (Base) = 100                        │   │  │
│   │  │                                               │   │  │
│   │  │   ┌─────────────────────────────────────┐   │   │  │
│   │  │   │ 修饰器组 (Modifiers)                 │   │   │  │
│   │  │   │                                       │   │   │  │
│   │  │   │   Add:     +50                       │   │   │  │
│   │  │   │   Mul:     0.2                       │   │   │  │
│   │  │   │   FinalAdd: +10                      │   │   │  │
│   │  │   │                                       │   │   │  │
│   │  │   └─────────────────────────────────────┘   │   │  │
│   │  │                   │                         │   │  │
│   │  │                   ▼                         │   │  │
│   │  │   (100 + 50) * (1 + 0.2) + 10 = 190       │   │  │
│   │  │                                               │   │  │
│   │  └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **NumberValue** | 带修饰器的数值容器 | 带加成的计算器 |
| **NumberModifier** | 对数值的修改 | 计算器的输入按钮 |
| **NumberModifierHandle** | 修饰器的"撤销凭证" | 取消按钮的编号 |
| **NumberEffect** | 多个修饰器的组合包 | 一套组合键 |
| **NumberValueMode** | 计算模式 | 计算公式选择 |

## 四、核心组件详解

### 4.1 NumberValue - 数值容器

**通俗解释**：NumberValue 是"带加成的计算器"，管理基础值和多个修饰器。

```
┌─────────────────────────────────────────────────────────────┐
│                    NumberValue 结构                          │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public sealed class NumberValue                     │  │
│   │   {                                                  │  │
│   │       public NumberValueMode Mode { get; }           │  │
│   │       public float BaseValue { get; set; }           │  │
│   │       public float Value { get; }                    │  │
│   │                                                       │  │
│   │       public NumberModifierHandle Apply(...);         │  │
│   │       public bool Remove(...);                         │  │
│   │       public void Clear(int sourceId = 0);            │  │
│   │       public NumberEffectHandle ApplyEffect(...);     │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   创建方式：                                          │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   // 基础值模式（无修饰器）                           │  │
│   │   var v1 = new NumberValue(NumberValueMode.BaseOnly);│  │
│   │                                                       │  │
│   │   // 加法模式                                        │  │
│   │   var v2 = new NumberValue(NumberValueMode.BaseAdd);│  │
│   │                                                       │  │
│   │   // 乘算模式（常用）                                 │  │
│   │   var v3 = new NumberValue(NumberValueMode.BaseAddMul, 100f);│  │
```

### 4.2 NumberValueMode - 计算模式

**通俗解释**：Mode 是"计算公式选择器"，决定最终值如何计算。

```
┌─────────────────────────────────────────────────────────────┐
│                    NumberValueMode 枚举                      │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public enum NumberValueMode                        │  │
│   │   {                                                  │  │
│   │       BaseOnly = 0,      // 只返回基础值            │  │
│   │       BaseAdd = 1,       // Base + Add + FinalAdd   │  │
│   │       BaseAddMul = 2,    // (Base+Add)*(1+Mul)+FA  │  │
│   │       OverrideOnly = 3    // Override 或 Base        │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   BaseOnly                                           │  │
│   │   ───────────                                        │  │
│   │   return Base;                                       │  │
│   │   // 修饰器被忽略                                    │  │
│   │                                                       │  │
│   │   BaseAdd                                            │  │
│   │   ────────                                           │  │
│   │   v = Base + Add + FinalAdd;                        │  │
│   │   if (HasOverride) v = Override;                     │  │
│   │   return v;                                          │  │
│   │                                                       │  │
│   │   BaseAddMul (常用)                                  │  │
│   │   ────────────────                                    │  │
│   │   v = (Base + Add) * (1 + Mul) + FinalAdd;          │  │
│   │   if (HasOverride) v = Override;                     │  │
│   │   return v;                                          │  │
│   │                                                       │  │
│   │   OverrideOnly                                        │  │
│   │   ─────────────                                      │  │
│   │   return HasOverride ? Override : Base;              │  │
│   │   // 没有加成，只有覆盖                               │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   伤害计算示例（BaseAddMul）：                        │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   Base = 100 (基础伤害)                              │  │
│   │   Add = 50 (装备加成 +50)                            │  │
│   │   Mul = 0.2 (技能加成 +20%)                          │  │
│   │   FinalAdd = 10 (最终加成)                            │  │
│   │                                                       │  │
│   │   计算：                                              │  │
│   │   = (100 + 50) * (1 + 0.2) + 10                     │  │
│   │   = 150 * 1.2 + 10                                   │  │
│   │   = 180 + 10                                          │  │
│   │   = 190                                               │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 NumberModifier - 修饰器

**通俗解释**：Modifier 是"对数值的修改"，包含操作类型、数值和来源。

### 4.5 Handle 模式

**通俗解释**：Handle 是"修饰器的遥控器"，用完可以精准移除。

```
┌─────────────────────────────────────────────────────────────┐
│                    Handle 模式                              │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public readonly struct NumberModifierHandle        │  │
│   │   {                                                  │  │
│   │       public readonly int Value;                    │  │
│   │       public bool IsValid => Value != 0;            │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   使用流程：                                          │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   1️⃣ 应用修饰器，获得 Handle                        │  │
│   │   var handle = number.Apply(new NumberModifier(     │  │
│   │       NumberModifierOp.Add, 50f));                  │  │
│   │                                                       │  │
│   │   2️⃣ 使用数值                                        │  │
│   │   var result = number.Value;  // 已经包含加成       │  │
│   │                                                       │  │
│   │   3️⃣ 移除修饰器                                      │  │
│   │   number.Remove(handle);  // 用 Handle 移除         │  │
│   │                                                       │  │
│   │   4️⃣ 再次使用数值                                    │  │
│   │   var clean = number.Value;  // 加成已移除          │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   Handle 无效检查：                                   │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   if (!handle.IsValid) {                            │  │
│   │       // Handle 无效，不处理                         │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   │   // 或者用 null 检查：                              │  │
│   │   if (handle != null) {                             │  │
│   │       number.Remove(handle);                        │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Dirty Flag Pattern（脏标记模式）                     │
│      ─────────────────────────────────────                  │
│      值变化时标记脏，读取时才重新计算                        │
│      避免每次修改都重新计算                                  │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   Apply/Remove → _dirty = true                  │   │
│      │   Value getter → if (_dirty) Recompute()        │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Handle Pattern（句柄模式）                            │
│      ─────────────────────────────────────                  │
│      修饰器用句柄管理，隐藏内部索引                          │
│      支持 O(1) 移除                                          │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   Apply → return Handle(handle)                  │   │
│      │   Remove(handle) → O(1) 查找并移除              │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ RAII Pattern（资源获取即初始化）                     │
│      ─────────────────────────────────────                  │
│      EffectHandle 实现 IDisposable                          │
│      using 语句块自动释放资源                                │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   using (damage.ApplyEffect(buff)) {            │   │
│      │       // buff 生效                               │   │
│      │   } // 离开时自动移除                            │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Immutable Struct（不可变结构体）                      │
│      ─────────────────────────────────────                  │
│      NumberModifier 是只读结构体                            │
│      线程安全，可以自由传递                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Dirty Flag | 避免重复计算 |
| Handle | O(1) 修饰器管理 |
| RAII | 自动资源释放 |
| Immutable | 线程安全，自由传递 |

## 六、使用指南

### 6.1 基础伤害计算示例

```csharp
// 场景：MOBA 伤害计算

public class DamageCalculator
{
    // 伤害 = (基础伤害 + 固定加成) * (1 + 百分比加成) + 最终加成
    public float CalculateDamage(
        float baseDamage,
        float flatBonus,
        float percentBonus,
        float finalBonus)
    {
        var damage = new NumberValue(NumberValueMode.BaseAddMul, baseDamage);

        // 添加固定加成
        damage.Apply(new NumberModifier(NumberModifierOp.Add, flatBonus, sourceId: 1));

        // 添加百分比加成
        damage.Apply(new NumberModifier(NumberModifierOp.Mul, percentBonus, sourceId: 2));

        // 添加最终加成
        damage.Apply(new NumberModifier(NumberModifierOp.FinalAdd, finalBonus, sourceId: 3));

        return damage.Value;
    }
}

// 使用
var calculator = new DamageCalculator();
var damage = calculator.CalculateDamage(
    baseDamage: 100f,
    flatBonus: 50f,      // +50 固定伤害
    percentBonus: 0.2f,  // +20% 伤害
    finalBonus: 10f      // +10 最终伤害
);
// 结果 = (100 + 50) * (1 + 0.2) + 10 = 190
```

### 6.2 Buff 效果系统示例

```csharp
// 场景：技能Buff效果

public class BuffSystem
{
    // Buff 效果定义
    public static NumberEffect PowerBuff(int buffId) =>
        new NumberEffect(
            new NumberEffect.Entry(new NumberModifier(NumberModifierOp.Add, 50f, buffId)),
            new NumberEffect.Entry(new NumberModifier(NumberModifierOp.Mul, 0.25f, buffId))
        );

    public static NumberEffect ArmorDebuff(int debuffId) =>
        new NumberEffect(
            new NumberEffect.Entry(new NumberModifier(NumberModifierOp.Mul, -0.3f, debuffId))
        );

    // 应用 buff
    public NumberEffectHandle ApplyBuff(NumberValue target, NumberEffect effect)
    {
        return target.ApplyEffect(effect);
    }
}

// 使用
var damage = new NumberValue(NumberValueMode.BaseAddMul, 100f);
var buffSystem = new BuffSystem();

// 应用力量buff
var buffHandle = buffSystem.ApplyBuff(damage, BuffSystem.PowerBuff(buffId: 1));

Console.WriteLine($"buff中: {damage.Value}");  // (100+50)*(1+0.25) = 187.5

// buff 结束
buffHandle.Dispose();
Console.WriteLine($"buff后: {damage.Value}");  // 100
```

### 6.4 复杂战斗计算示例

```csharp
// 场景：完整的伤害计算流程

public float CalculateFinalDamage(AttackInfo attack, TargetInfo target, List<BuffEffect> activeBuffs)
{
    // 创建伤害数值
    var damage = new NumberValue(NumberValueMode.BaseAddMul, attack.BaseDamage);

    // 应用攻击者buff
    foreach (var buff in attack.Buffs)
    {
        damage.Apply(new NumberModifier(buff.Op, buff.Value, buff.SourceId));
    }

    // 应用目标debuff
    foreach (var debuff in target.Debuffs)
    {
        damage.Apply(new NumberModifier(debuff.Op, debuff.Value, debuff.SourceId));
    }

    // 应用护甲减免
    var armorReduction = target.Armor / (target.Armor + 100f);
    damage.Apply(new NumberModifier(NumberModifierOp.Mul, -armorReduction, sourceId: 0));

    // 应用暴击
    if (target.IsCritical)
    {
        damage.Apply(new NumberModifier(NumberModifierOp.Mul, target.CritMultiplier - 1f, sourceId: 0));
    }

    // 返回最终伤害
    return damage.Value;
}
```

| 技能加成 | 数值系统 | 临时计算，用完即弃 |
| 装备属性 | 属性系统 | 持久存储，需要注册 |
| Buff效果 | 数值系统 | 临时应用，可组合 |
| 伤害计算 | 数值系统 | 中间过程，轻量高效 |
| 护甲减免 | 数值系统 | 临时计算，结果存储 |

## 八、解决的问题与适用场景

### 8.1 数值系统模块解决什么问题

```
┌─────────────────────────────────────────────────────────────┐
│                    数值系统解决的问题                         │
│                                                             │
│   1️⃣ 轻量计算                                            │
│      ─────────────────────────────────────                  │
│      不需要全局注册，即用即创建                             │
│      用完即弃，不占用持久存储                              │
│                                                             │
│   2️⃣ 灵活加成                                            │
│      ─────────────────────────────────────                  │
│      支持多种加成类型，可自由组合                          │
│      Base + Add + Mul + FinalAdd 公式                      │
│                                                             │
│   3️⃣ 批量管理                                            │
│      ─────────────────────────────────────                  │
│      Effect 支持批量应用和移除                             │
│      Handle 确保不漏移除                                    │
│                                                             │
│   4️⃣ 安全计算                                            │
│      ─────────────────────────────────────                  │
│      NaN/Inf 检查防止异常值                                │
│      脏标记避免重复计算                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
