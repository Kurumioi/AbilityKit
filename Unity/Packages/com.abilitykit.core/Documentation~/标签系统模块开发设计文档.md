# Ability-Kit 标签系统模块开发设计文档

> **阅读对象**：希望了解标签系统如何设计的开发者
>
> **文档目标**：让你理解"标签如何组织成层级结构"、"标签匹配规则如何工作"、"标签如何驱动游戏逻辑"

## 一、设计理念：为什么需要标签系统？

### 1.1 传统做法的问题

```
❌ 传统标签管理的问题：

1. 字符串硬编码
   ┌─────────────────────────────────────────────────────────┐
   │ // 用字符串或枚举表示标签                           │
   │ public const string TAG_ENEMY = "Enemy";              │
   │ public const string TAG_BOSS = "Boss";                │
   │                                                          │
   │ if (gameObject.tag == "Enemy") { ... }                │
   │                                                          │
   │ 问题：字符串容易写错，IDE 无法提示                    │
   │ 问题：层级关系难以表达                                │
   └─────────────────────────────────────────────────────────┘

2. 枚举爆炸
   ┌─────────────────────────────────────────────────────────┐
   │ public enum ObjectType {                               │
   │     Player,                                           │
   │     Enemy,                                            │
   │     Boss,                                             │
   │     NPC,                                              │
   │     Building,                                         │
   │     // ... 需要组合时怎么办？                        │
   │ }                                                      │
   │                                                          │
   │ 问题：枚举难以组合，层级关系难以维护                  │
   └─────────────────────────────────────────────────────────┘

3. 层级标签混乱
   ┌─────────────────────────────────────────────────────────┐
   │ // 用层级路径表示                                      │
   │ "Enemy.Boss.Dragon"                                   │
   │ "Enemy.Boss.SkeletonKing"                              │
   │                                                          │
   │ 问题：需要手动解析字符串                              │
   │ 问题：匹配规则不明确                                  │
   └─────────────────────────────────────────────────────────┘

4. 标签变化难以追踪
   ┌─────────────────────────────────────────────────────────┐
   │ // 不知道谁添加/移除了什么标签                        │
   │ entity.AddTag("Stunned");                              │
   │ entity.RemoveTag("Stunned");                           │
   │                                                          │
   │ 问题：标签变化没有通知机制                            │
   │ 问题：难以调试和追踪                                  │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 标签系统的解决方案

```
✅ 标签系统的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   标签 = 带 ID 的层级字符串                                │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  传统方式：                                          │  │
│   │  ┌───────────────────────────────────────────┐    │  │
│   │   │  "Enemy" ──► 字符串                      │    │  │
│   │   │  "Boss"  ──► 字符串                      │    │  │
│   │   │  "Dragon" ──► 字符串                     │    │  │
│   │   │                                              │    │  │
│   │   │  关系：手动维护                             │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  标签系统方式：                                      │  │
│   │  ┌───────────────────────────────────────────┐    │  │
│   │   │  GameplayTag { Id = 1, Name = "Enemy" } │    │  │
│   │   │      │                                     │    │  │
│   │   │      ├── GameplayTag { Id = 2,           │    │  │
│   │   │      │     Name = "Enemy.Boss" }          │    │  │
│   │   │      │       │                            │    │  │
│   │   │      │       └── GameplayTag { Id = 3,   │    │  │
│   │   │      │             Name = "Enemy.Boss.Dragon" }│  │
│   │   │      │                                     │    │  │
│   │   │      └── GameplayTag { Id = 4,           │    │  │
│   │   │            Name = "Enemy.Minion" }        │    │  │
│   │   │                                              │    │  │
│   │   │  关系：自动继承                              │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   好处：                                                    │
│   - 层级关系自动管理                                      │
│   - 匹配规则明确（父标签匹配子标签）                      │
│   - 变化可追踪                                            │
│   - IDE 友好（静态生成代码）                              │
│                                                             │

```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    标签系统的三大承诺                          │
│                                                             │
│   1️⃣ 层级                                              │
│      标签天然是树形结构，父子关系自动继承                   │
│                                                             │
│   2️⃣ 匹配                                            │
│      灵活的匹配规则，支持精确匹配和继承匹配                 │
│                                                             │
│   3️⃣ 可追踪                                            │
│      完整的变化通知，便于调试和逻辑驱动                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解标签系统

### 2.1 什么是"GameplayTag"？

**通俗解释**：GameplayTag 是"带 ID 的层级路径"，类似文件系统中的路径。

```
┌─────────────────────────────────────────────────────────────┐
│                    GameplayTag = ID + 名称                   │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  标签示例                                            │  │
│   │  ─────────────────────────────────────────────      │  │
│   │                                                       │  │
│   │  Tag("Event.Damage.Physical")                       │  │
│   │                                                       │  │
│   │  ┌───────────────────────────────────────────────┐  │  │
│   │   │  Name: "Event.Damage.Physical"              │   │  │
│   │   │  Id: 5                                        │   │  │
│   │   │  Parent: "Event.Damage" (Id=4)              │   │  │
│   │   │  GrandParent: "Event" (Id=1)                │   │  │
│   │   └───────────────────────────────────────────────┘  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  层级树                                              │  │
│   │  ───────────────────────────────────────            │  │
│   │                                                       │  │
│   │  Root (Id=0)                                        │  │
│   │  └─► Event (Id=1)                                  │  │
│   │      └─► Damage (Id=4)                              │  │
│   │          ├─► Physical (Id=5)                       │  │
│   │          └─► Magic (Id=6)                          │  │
│   │  └─► State (Id=2)                                  │  │
│   │      └─► Stunned (Id=7)                            │  │
│   │      └─► Invincible (Id=8)                         │  │
│   │  └─► Type (Id=3)                                   │  │
│   │      ├─► Enemy (Id=9)                              │  │
│   │      └─► Ally (Id=10)                              │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **GameplayTag** | 标签 | 文件路径 |
| **GameplayTagContainer** | 标签容器 | 文件夹 |
| **GameplayTagManager** | 标签管理器 | 文件系统 |
| **GameplayTagRequirements** | 标签需求 | 权限要求 |
| **GameplayTagDelta** | 标签变化 | 文件变更记录 |
| **TagSource** | 标签来源 | 操作者身份 |

### 4.1 GameplayTag - 标签结构体

**通俗解释**：GameplayTag 是"带 ID 的标签"，只读且高效。

```csharp

│   │                                                       │  │

│   │   {                                                  │  │
│   │       internal readonly int Id;                      │  │
│   │                                                       │  │
│   │       public int Value => Id;                        │  │
│   │       public string Name => ...;                     │  │

│   │   }                                                  │  │
│   │                                                       │  │

│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   核心方法：                                          │  │
```

### 4.5 GameplayTagDelta - 标签变化

**通俗解释**：GameplayTagDelta 是"标签变更记录"，记录一次变化中新增和移除的标签。

```csharp
┌─────────────────────────────────────────────────────────────┐
│                    GameplayTagDelta 结构                        │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │

│                                                             │

│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                              │   │  │
│   │   │  监听标签变化时，同时获取：                   │   │  │
│   │   │                                              │   │  │
│   │   │  - 当前所有标签 (currentTags)                │   │  │
│   │   │  - 新增的标签 (delta.Added)                  │   │  │
│   │   │  - 移除的标签 (delta.Removed)                │   │  │
│   │   │                                              │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Flyweight Pattern（享元模式）                         │
│      ───────────────────────────────────────────────        │
│      标签通过 ID 共享，同名标签是同一个实例                   │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   Tag("Event.Damage") ──► GameplayTag { Id=5 }  │   │
│      │   Tag("Event.Damage") ──► GameplayTag { Id=5 }  │   │
│      │                                                   │   │
│      │   两次调用返回相同的 GameplayTag                  │   │
│      │   只存储一份 Node 数据                           │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Singleton Pattern（单例模式）                         │
│      ───────────────────────────────────────────────        │
│      GameplayTagManager 是全局单例                          │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   public sealed class GameplayTagManager         │   │
│      │   {                                              │   │
│      │       public static Manager Instance { get; }    │   │
│      │   }                                              │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Iterator Pattern（迭代器模式）                        │
│      ───────────────────────────────────────────────        │
│      GameplayTagContainer 实现 IEnumerable                  │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   foreach (var tag in container) { ... }        │   │
│      │                                                   │   │
│      │   container.GetEnumerator()                     │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Observer Pattern（观察者模式）                         │
│      ───────────────────────────────────────────────        │
│      ITagChangeSubscriber 订阅标签变化                      │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   interface ITagChangeSubscriber {              │   │
│      │       void OnTagsChanged(ownerId, delta, ...); │   │
│      │   }                                              │   │
│      │                                                   │   │
│      │   ITagEffectRouter.Register(subscriber);        │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   5️⃣ Command Pattern（命令模式）                            │
│      ───────────────────────────────────────────────        │
│      TagTemplateRuntime 封装标签变更命令                    │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   TagTemplateRuntime {                          │   │
│      │       GrantTags:   添加这些标签                 │   │
│      │       RemoveTags:  移除这些标签                 │   │
│      │   }                                              │   │
│      │                                                   │   │
│      │   ApplyTemplate() ──► 执行模板                 │   │
│      │                                                   │   │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 标签匹配示例

```csharp
// 场景：检查实体是否属于某种类型

public bool IsEnemy(Entity entity)
{
    return entity.Tags.HasTag(GameplayTags.Tag("Type.Enemy"));
}

public bool CanBeTargeted(Entity entity)
{
    // 不能是友方，不能是无敌状态
    if (entity.Tags.HasAny(_blockedTags)) return false;
    if (!entity.Tags.HasAll(_requiredTags)) return false;
    return true;
}

// 场景：检查标签是否匹配技能条件

{
    return skill.Requirements.IsSatisfiedBy(caster.Tags);
}
```

### 6.3 标签变化监听

```csharp
// 场景：监听实体标签变化

public class TagListener : ITagChangeSubscriber
{
    public void OnTagsChanged(int ownerId, GameplayTagContainer currentTags,
        GameplayTagDelta delta, TagSource source)
    {
        // 处理新增的标签
        foreach (var tag in delta.Added)
        {
            if (tag.IsParentOf(GameplayTags.Tag("State.Stunned")))
            {
                OnStunApplied(ownerId, source);
            }
        }

        // 处理移除的标签
        foreach (var tag in delta.Removed)
        {
            if (tag == GameplayTags.Tag("Buff.Invisible"))
            {
                OnInvisibleRemoved(ownerId, source);
            }
        }
    }
}

// 注册监听
var router = GetTagEffectRouter();
router.Register(new TagListener());
```

### 6.4 标签模板应用

```csharp
// 场景：应用标签模板（如 Buff 效果）

public class BuffSystem
{
    private IGameplayTagService _tagService;
    private ITagTemplateRegistry _templateRegistry;

    public void ApplyBuff(int targetId, int buffTemplateId, TagSource source)
    {
        if (!_templateRegistry.TryGet(buffTemplateId, out var template))
        {
            return;
        }

        // 检查条件
        var currentTags = _tagService.GetTags(targetId);
        if (!template.Requirements.IsSatisfiedBy(currentTags))
        {
            return;
        }

        // 应用模板
        _tagService.ApplyTemplate(targetId, buffTemplateId, source);
    }
}
```
