# AbilityKit Pipeline Runtime 目录职责

`Runtime` 目录按运行时职责拆分，新增代码应优先放入对应边界内，避免把构建、执行、生命周期和诊断逻辑混在同一层级。

## Core

- `Core/Pipeline`：管线运行核心，包含普通管线、即时管线、复合阶段调度与扩展点触发。
- `Core/Time`：运行时可替换时间源。

## Graph

`Graph` 只负责图式/DSL 构建入口，把阶段节点组合成可执行结构，不直接持有运行态，也不访问生命周期注册表。

## Phase

- `Phase/Core`：阶段基类和阶段运行通用能力。
- `Phase/Composite`：顺序、并行等组合阶段。
- `Phase/Condition`：条件节点、条件组合和条件阶段。
- `Phase/Timing`：延迟、重复、时间线等时间相关阶段。

阶段目录只表达“如何执行一个阶段”，不负责创建完整管线实例、不负责全局注册、不负责调试输出落盘。

## Lifecycle

`Lifecycle` 负责活跃运行实例的注册、查询、中断和生命周期观测契约。高频查询应优先使用填充式 API 或租约 API，避免临时分配。

## Data

`Data` 放置跨模块共享的数据结构、结果模型、快照模型和轻量标识值。

## Debug

`Debug` 放置调试钩子、追踪记录器与追踪事件数据，不参与核心执行决策。

## Pooling

`Pooling` 集中管理 Pipeline 包内部对象池键、默认配置和租借/归还入口。
