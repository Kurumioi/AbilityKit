## 1. 概述

### 1.1 设计目标

本设计文档描述 AbilityKit 触发器框架中的**统一调度管理系统**，用于：

1. **统一时间调度**：为 Buff、子弹、AOE、Trigger 等需要周期执行的模块提供统一的时间调度基础设施
2. **灵活选择**：根据业务需求选择合适的调度管理器
3. **行为复用**：通过适配器复用现有的 Behavior 调度层（`ISchedulableBehavior`）逻辑
4. **可扩展性**：支持自定义调度策略和调度效果

### 1.2 核心问题

在游戏开发中，以下场景都需要周期性的时间调度：

| 场景 | 需求 | 推荐管理器 |
|-----|------|-----------|
| Buff 系统 | 周期伤害、周期治疗、持续增益/减益效果 | SimpleScheduleManager |
| 子弹系统 | 子弹飞行轨迹更新、碰撞检测 | SimpleScheduleManager |
| AOE 系统 | 区域持续伤害、范围检测 | SimpleScheduleManager |
| Trigger 系统 | 触发器内 Action 的生命周期管理、按 Trigger 分组控制 | GroupedScheduleManager |

### 3.4 ScheduleContext（调度上下文）

```csharp
public readonly struct ScheduleContext
{
    public readonly int InstanceId;
    public readonly int ContextId;
    public readonly float DeltaTimeMs;
    public readonly float ElapsedMs;
    public readonly float ScaledDeltaMs;
    public readonly float IntervalMs;
    public readonly int ExecutionCount;
    public readonly int MaxExecutions;
    public readonly float Speed;
    public readonly string InterruptReason;

    // 计算属性
    public bool IsMaxExecutionsReached => MaxExecutions > 0 && ExecutionCount >= MaxExecutions;
    public float TimeToNextExecute => IntervalMs > 0 ? IntervalMs - (ElapsedMs % IntervalMs) : 0;
    public float Progress => MaxExecutions > 0 ? (float)ExecutionCount / MaxExecutions : 0;
}
```

## 6. 集成示例

### 6.1 Buff 系统集成（SimpleScheduleManager）

```csharp
public class BuffScheduleManager
{
    private readonly IScheduleManager _manager;
    private readonly IScheduleEffectFactory _effectFactory;
    private readonly IBehaviorContextFactory _contextFactory;

    public BuffScheduleManager()
    {
        // Buff 系统使用 SimpleScheduleManager
        _manager = new SimpleScheduleManager();
    }

    /// <summary>
    /// 应用周期性 Buff
    /// </summary>
    public ScheduleHandle ApplyPeriodicBuff(Buff buff, BuffConfig config)
    {
        // 1. 创建上下文
        var context = _contextFactory.CreateContext(new
        {
            BuffId = buff.Id,
            TargetId = buff.TargetId,
            CasterId = buff.CasterId
        });

        // 2. 创建调度效果
        var effect = _effectFactory.CreateBuffPeriodicDamage(
            buff.Id,
            new BuffPeriodicConfig
            {
                TriggerId = config.TriggerId,
                DamagePerTick = config.DamagePerTick,
                PeriodMs = config.PeriodMs,
                DurationMs = config.DurationMs,
                MaxExecutions = config.MaxExecutions
            },
            context
        );

        // 3. 注册调度（使用 ContextId 关联）
        return _manager.RegisterPeriodic(
            intervalMs: (float)config.PeriodMs,
            maxExecutions: config.MaxExecutions,
            contextId: buff.Id,  // ← 关联到具体 Buff
            effect
        );
    }

    /// <summary>
    /// 移除 Buff
    /// </summary>
    public void RemoveBuff(Buff buff)
    {
        if (buff.ScheduleHandle.IsValid)
        {
            _manager.Cancel(buff.ScheduleHandle);
            buff.ScheduleHandle = ScheduleHandle.Invalid;
        }
    }

    /// <summary>
    /// 更新调度（每帧调用）
    /// </summary>
    public void Update(float deltaTimeMs)
    {
        _manager.Update(deltaTimeMs);
    }
}
```

### 6.2 Trigger 系统集成（GroupedScheduleManager）

```csharp
public class TriggerScheduleManager
{
    private readonly IGroupedScheduleManager _manager;

    public TriggerScheduleManager()
    {
        // Trigger 系统使用 GroupedScheduleManager
        _manager = new GroupedScheduleManager();
    }

    /// <summary>
    /// 触发器激活
    /// </summary>
    public void OnTriggerActivated(int triggerId)
    {
        _manager.OnGroupActivated(triggerId);
    }

    /// <summary>
    /// 触发器停用（批量中断所有相关调度）
    /// </summary>
    public void OnTriggerDeactivated(int triggerId)
    {
        _manager.InterruptGroup(triggerId, "Trigger deactivated");
        _manager.OnGroupDeactivated(triggerId);
    }

    /// <summary>
    /// 注册调度（按 Trigger 分组）
    /// </summary>
    public ScheduleHandle RegisterForTrigger(int triggerId, ScheduleRegisterRequest request, IScheduleEffect effect)
    {
        return _manager.RegisterForGroup(triggerId, request, effect);
    }

    /// <summary>
    /// 暂停所有属于某 Trigger 的调度
    /// </summary>
    public void PauseTrigger(int triggerId)
    {
        _manager.PauseGroup(triggerId);
    }

    /// <summary>
    /// 恢复所有属于某 Trigger 的调度
    /// </summary>
    public void ResumeTrigger(int triggerId)
    {
        _manager.ResumeGroup(triggerId);
    }

    /// <summary>
    /// 获取某 Trigger 的调度项数量
    /// </summary>
    public int GetScheduleCount(int triggerId)
    {
        return _manager.GetItemCountByGroup(triggerId);
    }

    /// <summary>
    /// 更新调度（每帧调用）
    /// </summary>
    public void Update(float deltaTimeMs)
    {
        _manager.Update(deltaTimeMs);
    }
}
```

## 7. 设计决策

### 7.1 分层接口设计

**决策**：将调度管理器分为 `IScheduleManager` 和 `IGroupedScheduleManager` 两层接口

**理由**：
1. **按需选择**：Buff/子弹/AOE 不需要分组能力，使用 `SimpleScheduleManager` 更简洁
2. **职责清晰**：每个接口职责明确，不会混淆
3. **扩展性**：未来可以新增其他类型的调度管理器（如按场景分组）

### 7.2 为什么不统一用 GroupedScheduleManager

**问题**：为什么不统一使用 `GroupedScheduleManager`，而要提供两个实现？

**回答**：
1. **简单性**：Buff 等场景不需要分组能力，使用 `SimpleScheduleManager` 更直观
2. **性能**：虽然差异不大，但 `SimpleScheduleManager` 没有分组索引，内存占用略低
3. **语义**：接口名称传达了明确的用途，减少使用者困惑

**注意**：`GroupedScheduleManager` 可以替代 `SimpleScheduleManager` 使用，反之不行。

### 7.3 ContextId vs GroupId

**决策**：`ContextId` 用于业务对象关联，`GroupId`（即 `TriggerId`）用于批量控制

**理由**：
1. `ContextId`：关联到具体的业务对象（BuffId、子弹Id、AOEId）
2. `GroupId`：用于批量控制一组相关调度项

**示例**：
```csharp
// Buff 调度
manager.RegisterPeriodic(
    contextId: buff.Id,    // 关联到具体 Buff
    effect: buffEffect
);

// Trigger 调度
manager.RegisterPeriodicForGroup(
    groupId: triggerId,     // 按 Trigger 分组
    contextId: actionId,    // 也关联到具体 Action
    effect: actionEffect
);
```

| `Terminated` | 已终止（完成或中断） |

## 附录 C: 调度模式

| 模式 | 说明 | 使用场景 |
|-----|------|---------|
| `Immediate` | 立即执行一次 | 即时触发效果 |
| `Delayed` | 延迟后执行一次 | 延迟生效的技能 |
| `Periodic` | 周期性执行 | 持续伤害、周期增益 |
