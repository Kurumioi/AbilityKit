# 触发器系统设计文档

## 结论
- 正式主线以 `TriggerPlan`、`PlannedTrigger`、`RuleScheduler`、`ActionScheduler`、`TriggerRunner` 为核心。
- `Runtime/Dispatcher`、`Runtime/Scheduler`、`Runtime/Executable`、`Runtime/Legacy` 只保留历史记录、显式失败或已删除痕迹。
- 示例和验收材料应优先描述正式主线，不再强调旧兼容路径。

- 新文档不要再以旧调度/旧派发命名为主。
- 若历史条目不再有参考价值，应直接删除而不是长期保留。
