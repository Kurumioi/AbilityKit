# 游戏技能系统本质抽象与实现映射

> 从自然语言描述出发，拆解为本质抽象（WHO/WHEN/WHAT/HOW），再映射到具体技术实现。
> 
> **核心理念**：Buff、主动技能、被动技能、光环效果本质上都是同一种东西，只是配置参数不同。

---

## 零、统一抽象模型

```
┌─────────────────────────────────────────────────────────────────────┐
│                      IEffectConfig (统一配置)                         │
│                                                                      │
│   WHO ──────────→ 目标是谁？                                      │
│    │            (target / enemies / allies / in_range)             │
│    │                                                              │
│    ▼                                                              │
│   WHEN ─────────→ 什么时候触发？                                  │
│    │            (active / passive / periodic / condition)           │
│    │                                                              │
│    ▼                                                              │
│   WHAT ─────────→ 做什么效果？                                   │
│    │            (damage / heal / buff / stun / modifier)          │
│    │                                                              │
│    ▼                                                              │
│   HOW ──────────→ 怎么执行？                                      │
│                (immediate / continuous / phased / channeled)         │
└─────────────────────────────────────────────────────────────────────┘
```

### 配置决定类型，而非类型决定配置

| 传统思维 | 统一抽象 |
|---------|---------|
| Buff是一种东西 | Buff只是一组特定配置 |
| 技能是一种东西 | 技能只是另一种配置组合 |
| 被动是一种东西 | 被动只是触发条件不同 |
| 光环是一种东西 | 光环只是范围+持续的配置 |

```
配置1：Trigger=Active, Duration=0, Effect=Damage
         ↓ 就是 "主动技能"

配置2：Trigger=Passive, Duration=5s, Effect=DoT
         ↓ 就是 "Buff"

配置3：Trigger=Aura, Range=10m, Duration=Infinite, Effect=Modifier
         ↓ 就是 "光环"

配置4：Trigger=OnDamaged, Effect=CounterDamage
         ↓ 就是 "被动反击"

它们都使用同一个 EffectRegistry.Start(config)！
```

## 一、完整映射链路

```
自然语言 ──────────→ 本质抽象 ──────────→ 技术实现
  ↓                    ↓                    ↓
"对生命值低于30%的   WHO: enemies_in_range  → GameplayTag + Range
 前方30米锥形范围内   WHEN: condition       → TriggerRegistry
 的敌人造成伤害"     WHAT: damage(100)     → Modifier/Actions
                    HOW: immediate         → Pipeline/Skill
```

## 二、自然语言 → 本质抽象

### 2.1 常见自然语言模式

| 自然语言模式 | WHO | WHEN | WHAT | HOW |
|-------------|-----|------|------|-----|
| "对目标造成伤害" | target | - | damage | immediate |
| "对前方X米敌人" | enemies_in_cone | - | - | - |
| "每5秒造成伤害" | - | periodic(5s) | damage | - |
| "生命值低于30%" | - | condition(health<30%) | - | - |
| "施放后3秒" | - | delayed(3s) | - | - |
| "持续5秒" | - | - | duration(5s) | continuous |
| "眩晕2秒" | - | - | stun(2s) | immediate |
| "可驱散" | - | - | tag(purgable) | - |

### 2.2 自然语言解析规则

```
规则1：施法者相关
─────────────────────────────────────
"我" / "自身" / "自己"        → self
"施法者" / "释放者"           → caster

规则2：目标相关
─────────────────────────────────────
"对目标" / "对敌方"           → target
"对敌人" / "对敌方单位"       → enemies
"对友军" / "对队友"           → allies
"对所有敌人"                   → all_enemies

规则3：范围描述
─────────────────────────────────────
"前方X米"                      → cone(range=X)
"周围X米"                      → circle(range=X)
"半径X米"                      → circle(range=X)
"直线X米"                      → line(range=X)
"范围内"                       → in_range

规则4：触发条件
─────────────────────────────────────
"施放时" / "使用时"           → trigger(activate)
"受到伤害时"                   → trigger(on_damaged)
"造成伤害时"                   → trigger(on_damage_dealt)
"击杀时"                       → trigger(on_kill)
"生命值低于X%"                 → condition(health<X%)
"每X秒"                        → periodic(Xs)
"施放后X秒"                    → delayed(Xs)

规则5：效果描述
─────────────────────────────────────
"造成X点伤害"                  → damage(X)
"治疗X点"                      → heal(X)
"眩晕X秒"                      → stun(Xs)
"减速X%"                       → slow(X%)
"添加buff"                     → apply_buff
"移除buff"                     → remove_buff
"不可驱散"                     → tag(not_purgable)
"持续X秒"                      → duration(Xs)
```

## 三、案例分析

### 案例1：普通攻击

```
自然语言：
"对当前目标造成物理伤害，伤害值等于100%攻击力"
```

**拆解**：
```
WHO     : target              → 当前选中的目标
WHEN    : activate           → 玩家施放
WHAT    : damage(scaling=1.0) → 伤害，效果值=攻击力×100%
HOW     : immediate          → 立即执行
```

**实现映射**：
```
技术实现需求：
├── 目标系统：获取当前目标
├── 伤害计算：基础伤害 = 攻击力 × 1.0
└── 执行方式：Pipeline 立即执行

代码示例：
```csharp
var skillConfig = SkillConfig.Create()
    .WithTarget(TargetType.CurrentTarget)
    .WithTrigger(TriggerType.Active)
    .WithEffect(new DamageEffect(
        DamageType.Physical,
        Magnitude.FromAttribute("AttackPower", scaling: 1.0)
    ))
    .WithExecution(ExecutionType.Immediate);
```
```

### 案例2：持续伤害 Debuff

```
自然语言：
"对目标施加中毒效果，持续5秒，每秒造成10点伤害，可驱散"
```

**拆解**：
```
WHO     : target              → 施法时的目标
WHEN    : activate           → 玩家施放
WHAT    : 
         - apply_buff(dot_poison)
         - damage_over_time(10, every=1s)
         - duration(5s)
         - tag(purgable)
HOW     : continuous          → 持续执行
```

**实现映射**：
```
技术实现需求：
├── 标签系统：标记中毒状态
├── 调度系统：每秒触发伤害
├── 生命周期：5秒自动结束
├── 修改器：添加"可驱散"标签
└── 事件：结束时移除debuff

代码示例：
```csharp
var debuffConfig = ContinuousConfig.Timed(5000, 1000)  // 5秒，每秒触发
    .WithTags("Debuff.Poison", "Debuff.Purgable")
    .WithOnTick(ctx =>
    {
        ctx.Target.TakeDamage(10, DamageType.Poison);
    })
    .WithOnEnd(ctx =>
    {
        ctx.Target.RemoveTag("Debuff.Poison");
    });
```
```

### 案例5：被动触发技能

```
自然语言：
"被动：受到伤害时，有20%几率对攻击者造成反击，造成50点伤害"
```

**拆解**：
```
WHO     : attacker           → 攻击者（自己受击时触发）
WHEN    : on_receiving_damage (被动触发)
         - probability(20%)  → 20%概率
WHAT    : damage(50)         → 反击伤害
HOW     : immediate          → 立即执行
         - passive          → 被动技能
```

**实现映射**：
```
技术实现需求：
├── 事件系统：监听"受到伤害"事件
├── 概率判定：20%触发几率
├── 目标选择：来源是攻击者
├── 执行方式：立即反击
└── 冷却控制：反击冷却

代码示例：
```csharp
var passiveConfig = PassiveSkillConfig.Create()
    .WithTrigger(TriggerType.OnDamaged)
    .WithCondition(ctx => Random.value < 0.2f)
    .WithEffect(ctx => new DamageEffect(
        DamageType.Physical,
        Magnitude.Fixed(50),
        target: ctx.Attacker  // 攻击者是目标
    ))
    .WithExecution(ExecutionType.Immediate)
    .WithCooldown(2f);
```
```

### 案例6：光环技能

```
自然语言：
"光环：为自己周围10米内的所有友军提供20%攻击速度加成，持续直到主动取消"
```

**拆解**：
```
WHO     : allies_in_range(self, range=10m, except_self)
WHEN    : activate           → 主动激活，持续存在
WHAT    : 
         - modifier(attack_speed +20%)
         - apply_buff(attack_speed_boost)
WHAT(2) : modifier(remove on exit)
HOW     : continuous         → 持续存在
         - infinite         → 永久
         - active           → 主动激活
```

**实现映射**：
```
技术实现需求：
├── 范围系统：持续检测周围友军
├── 修改器系统：添加攻击速度修改
├── 标签系统：标记增益状态
├── 进入/离开检测：进出范围时添加/移除效果
└── 生命周期：主动取消

代码示例：
```csharp
var auraConfig = ContinuousConfig.Infinite(1000)  // 每秒检测
    .WithTags("Aura.AttackSpeed")
    .WithApplicationTags("Buff.AttackSpeedBoost")
    .WithModifiers(
        ModifierData.PercentAdd("AttackSpeed", 0.2f)
    )
    .WithOnTick(ctx =>
    {
        // 获取范围内的友军
        var allies = ctx.Entity.GetAlliesInRange(10f);
        foreach (var ally in allies)
        {
            // 确保有增益标签
            if (!ally.HasTag("Buff.AttackSpeedBoost"))
            {
                ally.AddModifier(attackSpeedModifier, ctx.SourceId);
                ally.AddTag("Buff.AttackSpeedBoost");
            }
        }
    })
    .WithOnExit(ctx =>
    {
        // 移除所有友军的增益
        RemoveAuraEffects(ctx.Entity);
    });
```
```

### 案例7：条件触发多段技能

```
自然语言：
"如果目标生命值低于30%，施放终结技，造成300%伤害；否则造成100%基础伤害"
```

**拆解**：
```
WHO     : target
WHEN    : activate           → 玩家施放
WHAT    : 
         - condition(health<30%)
           → true:  damage(scaling=3.0)  // 终结技
           → false: damage(scaling=1.0)  // 普通伤害
HOW     : conditional        → 条件分支
         - immediate        → 立即执行分支
```

**实现映射**：
```
技术实现需求：
├── 条件判断：检查目标生命值
├── 分支执行：根据条件选择效果
└── 执行方式：条件通过后立即执行

代码示例：
```csharp
var executeConfig = SkillConfig.Create()
    .WithTarget(TargetType.CurrentTarget)
    .WithEffect(
        new ConditionalEffect(
            condition: ctx => ctx.Target.HealthPercent < 0.3f,
            onTrue: new DamageEffect(
                DamageType.Physical,
                Magnitude.FromAttribute("AttackPower", scaling: 3.0)
            ),
            onFalse: new DamageEffect(
                DamageType.Physical,
                Magnitude.FromAttribute("AttackPower", scaling: 1.0)
            )
        )
    );
```
```

### 案例8：叠加触发

```
自然语言：
"每次普通攻击叠加1层减速层数，叠加到5层时定身目标2秒"
```

**拆解**：
```
WHO     : target
WHEN    : on_dealing_damage (被动：造成伤害时)
WHAT    : 
         - add_stack(slow)           → 添加1层减速
         - stacks=5
           → stun(2s)                → 触发定身
HOW     : stacking
         - on_stack_max(5)          → 5层时触发
         - reset_stacks              → 触发后重置
```

**实现映射**：
```
技术实现需求：
├── 堆叠系统：追踪减速层数
├── 触发检测：层数达到阈值
├── 效果应用：定身目标
├── 层数重置：触发后清零
└── 冷却控制：触发后进入冷却

代码示例：
```csharp
var slowConfig = ContinuousConfig.Timed(10000, 0)
    .WithTags("Debuff.Slow")
    .WithStackable(true)
    .WithMaxStacks(5)
    .WithOnStackChange(ctx =>
    {
        // 更新减速效果
        var slowAmount = ctx.Stacks * 0.1f;  // 每层10%
        ctx.Target.ModifyAttribute("MoveSpeed", -slowAmount);
        
        // 检查是否达到触发阈值
        if (ctx.Stacks >= 5)
        {
            // 触发定身
            ctx.Target.ApplyStun(2f);
            ctx.ResetStacks();
        }
    })
    .WithOnEnd(ctx =>
    {
        // 移除所有减速效果
        ctx.Target.RemoveModifier("SlowEffect");
    });
```
```

### 案例9：多阶段技能

```
自然语言：
"第一段：冲刺到目标位置；第二段：挥砍对周围敌人造成100点伤害；第三段：获得3秒护盾"
```

**拆解**：
```
WHO     : 
         - phase_1: self
         - phase_2: enemies_in_range
         - phase_3: self
WHEN    : activate
WHAT    :
         - phase_1: dash(to_target)
         - phase_2: damage(100)
         - phase_3: shield(3s)
HOW     : phased
         - delay(0)      → 第一段立即
         - delay(0.5s)   → 第二段延迟0.5秒
         - delay(1s)      → 第三段延迟1秒
```

**实现映射**：
```
技术实现需求：
├── 移动系统：位移到目标
├── 范围检测：周围敌人检测
├── 效果系统：伤害+护盾
└── 阶段管线：顺序执行各阶段

代码示例：
```csharp
var comboConfig = ContinuousConfig.Timed(2000, 0)
    .WithTags("Skill.Combo")
    .WithPipeline(new AbilityPipeline()
        // 第一阶段：冲刺
        .AddPhase(new AbilityInstantPhase(ctx =>
        {
            ctx.Caster.DashTo(ctx.Target.Position);
        }))
        // 第二阶段：范围伤害
        .AddPhase(new AbilityDelayPhase(500))  // 等待冲刺完成
        .AddPhase(new AbilityInstantPhase(ctx =>
        {
            var enemies = ctx.Caster.GetEnemiesInRange(5f);
            foreach (var enemy in enemies)
            {
                enemy.TakeDamage(100);
            }
        }))
        // 第三阶段：获得护盾
        .AddPhase(new AbilityDelayPhase(500))
        .AddPhase(new AbilityInstantPhase(ctx =>
        {
            ctx.Caster.GainShield(ShieldType.Physical, 3f);
        }))
    );
```
```

## 四、抽象 → 技术实现对照表

### WHO 实现映射

| 抽象 | 技术实现 |
|------|---------|
| self | Entity.Self / Caster |
| target | Entity.GetCurrentTarget() |
| caster | SourceEntity |
| enemies | Filter by Team.Neutral && Alive |
| allies | Filter by Team.Same && Alive |
| in_range | Distance/Collision Check |
| enemies_in_cone | Cone Shape Query |
| has_tag | GameplayTagContainer.HasTag() |

### WHEN 实现映射

| 抽象 | 技术实现 |
|------|---------|
| activate | TriggerType.Active / PlayerInput |
| on_damaged | Event: OnDamaged / DamageReceived |
| on_kill | Event: OnKill / TargetDied |
| periodic | SchedulerRegistry.Periodic() |
| delayed | SchedulerRegistry.Delayed() |
| condition | TriggerPlan.When() / Predicate |
| stacks=5 | StackCounter == 5 |

### WHAT 实现映射

| 抽象 | 技术实现 |
|------|---------|
| damage(X) | Action: DealDamage / Modifier: -HP |
| heal(X) | Action: RestoreHealth / Modifier: +HP |
| stun(Xs) | ApplyTag("Stunned") / DisableMovement |
| slow(X%) | Modifier: MoveSpeed × (1-X%) |
| shield(Xs) | ShieldComponent.AddShield(duration=Xs) |
| apply_buff | ContinuousRegistry.Start() |
| DoT(X,every) | Periodic Damage Action |
| modifier(X) | ModifierData / AttributeModifier |

### HOW 实现映射

| 抽象 | 技术实现 |
|------|---------|
| immediate | PipelinePhase: Instant |
| delayed | PipelinePhase: Delay |
| channel | ContinuousConfig.Timed() + Interruptible |
| continuous | ContinuousConfig.Infinite() |
| phased | AbilityPipeline.AddPhase() |
| periodic | SchedulerRegistry.Periodic() |
| conditional | ConditionalPhase / If-Else |
| stacking | StackableContinuous / OnStackChange |

## 六、常见技能实现模板

### 模板1：单体伤害技能
```
抽象：target + activate + damage(X) + immediate
实现：SkillConfig → DamageAction → InstantPhase
```

### 模板2：持续伤害DOT
