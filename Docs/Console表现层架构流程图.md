# Console 表现层架构流程图

## 概述

本文档展示 Console Demo 表现层的完整数据流和组件关系，帮助理解**逻辑层**和**表现层**的正确边界。

## 二、输入处理流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           输入处理详细流程                                  │
│                                                                             │
│   1. 玩家输入                                                               │
│   ┌──────────────────────┐                                                │
│   │   键盘/命令           │                                                │
│   └──────────┬───────────┘                                                │
│              │                                                             │
│              ▼                                                             │
│   2. ConsoleInputFeature 收集                                              │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   - HudSkillClickSlot = 1  (技能槽位)                              │   │
│   │   - HudMoveDx/Dz        (移动向量)                                 │   │
│   │   - HudSkillAimSlot     (瞄准中的技能)                             │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   3. 转换为 PlayerInputCommand                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   Move 命令:    { OpCode: MobaOpCode.Move, Payload: {dx, dz} }    │   │
│   │   Skill 命令:   { OpCode: MobaOpCode.SkillInput, Payload: {...} } │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   4. DirectCallInputSink.Submit()                                         │
│              │                                                             │
│              ▼                                                             │
│   5. 逻辑层 MobaLobbyInputSink 处理                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   HandleMove():                                                     │   │
│   │     - 反序列化 Payload                                              │   │
│   │     - 获取 ActorEntity                                              │   │
│   │     - 添加 MoveInput Component                                      │   │
│   │                                                                    │   │
│   │   HandleSkillInput():                                              │   │
│   │     - 反序列化 SkillInputEvent                                      │   │
│   │     - 调用 SkillExecutor.HandleInput()                              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   6. 逻辑层执行计算                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   - 移动: ECS Systems 计算新位置                                    │   │
│   │   - 技能: SkillExecutor 执行技能 → 伤害计算 → 发布事件              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   7. 发布快照事件                                                          │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   - ActorTransformSnapshot   (位置更新)                            │   │
│   │   - DamageEventSnapshot      (伤害事件)                            │   │
│   │   - ProjectileEventSnapshot  (弹道事件)                           │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   8. 表现层渲染                                                            │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   ConsoleBattleViewEventSink.OnDamageEvent():                       │   │
│   │     - ShowFloatingText()    (显示伤害数字)                         │   │
│   │     - UpdateEntityHp()      (更新血条)                             │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、技能释放完整流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          技能释放完整流程                                   │
│                                                                             │
│   玩家按 '1' 键释放技能                                                         │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 1: ConsoleInputFeature                                        │   │
│   │                                                                       │   │
│   │   - 检测到技能槽位 1 被按下                                          │   │
│   │   - 设置 HudSkillClickSlot = 1                                     │   │
│   │   - 记录当前帧号                                                     │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 2: ConsoleInputHandler                                        │   │
│   │                                                                       │   │
│   │   - 检测到 HudSkillClickSlot > 0                                    │   │
│   │   - 创建 SkillInputEvent                                            │   │
│   │   - 发布到 LocalInputQueue                                          │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 3: DirectCallInputSink.Submit()                              │   │
│   │                                                                       │   │
│   │   - 将 PlayerInputCommand 转发到逻辑层                              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 4: MobaLobbyInputSink.HandleSkillInput()                     │   │
│   │                                                                       │   │
│   │   - 反序列化 SkillInputEvent                                        │   │
│   │   - 获取施法者 ActorEntity                                          │   │
│   │   - 调用 _skills.HandleInput(actorId, evt)                         │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 5: SkillExecutor.HandleInput()                                │   │
│   │                                                                       │   │
│   │   - 检查冷却状态                                                    │   │
│   │   - 检查目标有效性                                                  │   │
│   │   - 调用 MobaSkillPipeline 执行技能                                 │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 6: MobaEffectExecutionService.Execute()                       │   │
│   │                                                                       │   │
│   │   - 执行 TriggerPlan (技能效果配置)                                 │   │
│   │   - 计算伤害: baseDamage * skillMultiplier                          │   │
│   │   - 应用护甲减免: damage * (1 - defenseFactor)                      │   │
│   │   - 更新目标 HP                                                     │   │
│   │   - 判断是否死亡                                                    │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 7: 发布快照事件                                               │   │
│   │                                                                       │   │
│   │   MobaDamageEventSnapshotService.ReportDamage():                    │   │
│   │     - 记录: attackerId, targetId, damage, newHp, isDead            │   │
│   │                                                                       │   │
│   │   FrameSnapshotDispatcher.DispatchDamageEvent():                     │   │
│   │     - 分发 DamageEventData[]                                        │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 8: ConsoleBattleViewEventSink.OnDamageEvent()                 │   │
│   │                                                                       │   │
│   │   - 收到 DamageEventData[]                                          │   │
│   │   - _battleView.ShowFloatingText(targetId, $"-{damage}", ...)      │   │
│   │   - _battleView.UpdateEntityHp(targetId, newHp, maxHp)              │   │
│   │   - if (isDead) _battleView.ShowFloatingText(targetId, "DIED!")   │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│              │                                                             │
│              ▼                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  Step 9: ConsoleBattleView 渲染                                      │   │
│   │                                                                       │   │
│   │   - ASCII 字符显示: "Warrior [HP: 450/800] 位置 (10, 0, 5)"       │   │
│   │   - 伤害飘字: "-120" (红色)                                        │   │
│   │   - 死亡: "[X]" 标记                                               │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 六、快速检查清单

在表现层添加代码时，必须通过以下检查：

### 6.1 输入处理检查

```
□ 是否在 DirectCallInputSink 中添加了逻辑计算？
□ 是否在 ConsoleInputFeature 中计算伤害或修改 HP？
□ 是否直接调用了 SkillExecutor 而非通过 IWorldInputSink？

如果任何一项为 "是" → 重构到逻辑层
```

### 6.2 视图渲染检查

```
□ 是否在 ConsoleBattleViewEventSink 中计算伤害值？
□ 是否根据 HP 计算修改了 MaxHp 以外的值？
□ 是否直接访问了 ECS 世界或逻辑层服务？

如果任何一项为 "是" → 重构到事件携带完整状态
```

### 6.3 架构边界检查

```
□ 代码是否在 src/AbilityKit.Demo.Moba.Console/ 目录下？
□ 是否引用了 AbilityKit.Demo.Moba 命名空间的服务？
□ 是否实现了 IWorldInputSink 或 IBattleViewEventSink？

如果不是 → 检查是否放在正确的层
```

## 七、常见错误示例

### 7.1 错误：在表现层计算伤害

```csharp
// ❌ 错误：在 ConsoleBattleViewEventSink 中计算伤害
public void OnDamageEvent(...)
{
    // ❌ 错误：表现层不应该计算伤害
    var reducedDamage = damage * 0.8f;  // 护甲减免应该在逻辑层
    _battleView.ShowFloatingText(targetId, $"-{reducedDamage}", false);
}

// ✅ 正确：事件携带最终伤害值
public void OnDamageEvent(...)
{
    _battleView.ShowFloatingText(targetId, $"-{damage}", false);
}
```

### 7.2 错误：在表现层决定死亡

```csharp
// ❌ 错误：在表现层判断死亡
public void OnDamageEvent(...)
{
    _battleView.UpdateEntityHp(targetId, newHp, maxHp);
    if (newHp <= 0)  // ❌ 表现层不应该判断死亡
    {
        _battleView.ShowFloatingText(targetId, "DIED!", false);
    }
}

// ✅ 正确：事件携带 IsDead 标志
public void OnDamageEvent(...)
{
    _battleView.UpdateEntityHp(targetId, currentHp, maxHp);
    if (isDead)  // ✅ 逻辑层已判定
    {
        _battleView.ShowFloatingText(targetId, "DIED!", false);
    }
}
```

### 7.3 错误：在输入层直接处理逻辑

```csharp
// ❌ 错误：DirectCallInputSink 直接处理逻辑
public void Submit(FrameIndex frame, IReadOnlyList<PlayerInputCommand> inputs)
{
    foreach (var cmd in inputs)
    {
        if (cmd.OpCode == MobaOpCode.SkillInput)
        {
            // ❌ 错误：表现层不应该执行技能
            _skillExecutor.CastBySlot(...);
        }
    }
}

// ✅ 正确：只是转发，逻辑层处理
public void Submit(FrameIndex frame, IReadOnlyList<PlayerInputCommand> inputs)
{
    // ✅ 只是转发到逻辑层
    Platform.Log.Input($"[DirectCallInputSink] Submit {inputs.Count} commands");
}
```
