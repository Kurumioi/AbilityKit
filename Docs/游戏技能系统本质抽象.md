# 游戏技能系统 - 本质抽象

> 用元语言描述技能系统的本质需求，从「什么时候」「做什么」「怎么做」「对谁」四个维度拆解。

---

## 零、核心理念：一切皆为「效果」

```
传统思维：Buff、技能、被动、光环是四种不同的东西
         → 需要四套独立的系统来管理

统一思维：它们都是同一个东西的不同配置
         → 只需要一套系统，通过不同配置来区分
```

### 配置即类型

```
┌──────────────────────────────────────────────────────────────┐
│                    IEffectConfig                             │
│                                                              │
│   配置参数：                                                  │
│   - TriggerMode: Active      → 就是「主动技能」              │
│   - TriggerMode: Passive      → 就是「被动」                  │
│   - Duration: 5s            → 就是「有时间限制」            │
│   - Duration: Infinite       → 就是「光环」                   │
│   - Effect: Periodic Damage  → 就是「DoT」                   │
│                                                              │
│   结论：不需要为每种类型写专门的代码                         │
│         只需要一套统一的 IEffectConfig + EffectRegistry       │
└──────────────────────────────────────────────────────────────┘
```

### 代码示例

```csharp
// 传统做法：每个效果类型都有专门的代码
public class BuffSystem { ... }
public class SkillSystem { ... }
public class PassiveSystem { ... }
public class AuraSystem { ... }

// 统一做法：所有效果都是同一个接口的不同配置
public class EffectRegistry
{
    public ContinuousHandle Start(IEffectConfig config, EffectContext ctx)
    {
        // 统一处理所有类型
        return CreateAndStart(config, ctx);
    }
}

// 主动技能 = Trigger=Active, Duration=0
registry.Start(config.WithTrigger(TriggerMode.Active));

// 被动 = Trigger=Passive, Condition=OnDamaged
registry.Start(config.WithTrigger(TriggerMode.Passive));

// Buff = Trigger=Passive, Duration=5s
registry.Start(config.WithTrigger(TriggerMode.Passive).WithDuration(5f));

// 光环 = Trigger=Aura, Duration=Infinite, Range=10m
registry.Start(config.WithTrigger(TriggerMode.Aura).WithRange(10f));
```

## 一、本质四问

所有游戏技能系统都可以用这四个问题来描述：

```
┌─────────────────────────────────────────────────────────────┐
│                    技能本质四问                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   WHO ──────────→ 对谁？目标是谁？                         │
│    │                                                      │
│    │  施法者 / 目标 / 区域内的友军 / 敌人 / 自己          │
│    │                                                      │
│    ▼                                                      │
│   WHEN ─────────→ 什么时候？触发条件是什么？                │
│    │                                                      │
│    │  立即 / 延迟 / 周期 / 条件满足 / 事件触发           │
│    │                                                      │
│    ▼                                                      │
│   WHAT ─────────→ 做什么？产生什么效果？                   │
│    │                                                      │
│    │  伤害 / 治疗 / 添加Buff / 移除Buff / 移动 / ...     │
│    │                                                      │
│    ▼                                                      │
│   HOW ──────────→ 怎么做？执行方式是什么？                │
│    │                                                      │
│    │  立即 / 分段 / 引导 / 条件触发 / 叠加              │
│    │                                                      │
└─────────────────────────────────────────────────────────────┘
```

## 二、元语素定义

### WHO - 目标选择

```
元语素：目标
├── 选择器 (Selector)
│   ├── self           → 自己
│   ├── target         → 当前目标
│   ├── caster         → 施法者
│   ├── enemies        → 所有敌人
│   ├── allies         → 所有友军
│   ├── allies_except_self → 除自己外的友军
│   ├── in_range       → 范围内的单位
│   ├── has_tag       → 拥有特定标签的单位
│   └── filter        → 自定义过滤条件
│
└── 范围 (Range)
    ├── point          → 指定点
    ├── cone           → 扇形区域
    ├── line           → 直线区域
    ├── circle         → 圆形区域
    └── rectangle      → 矩形区域
```

**示例**：
```
技能A：对自己施放
技能B：对当前目标施放
技能C：对前方30米锥形范围内的所有敌人施放
技能D：对自己半径10米范围内的友军施放
```

### WHEN - 触发时机

```
元语素：时机
├── 主动触发
│   └── 玩家/AI显式施放
│
├── 被动触发
│   ├── 条件触发 (Condition)
│   │   ├── health_below(X)      → 生命值低于X%
│   │   ├── has_buff(X)         → 拥有X buff
│   │   ├── receives_damage     → 受到伤害
│   │   ├── deals_damage        → 造成伤害
│   │   └── cooldown_ready       → 冷却完毕
│   │
│   ├── 事件触发 (Event)
│   │   ├── on_hit              → 命中时
│   │   ├── on_kill             → 击杀时
│   │   ├── on_death            → 死亡时
│   │   ├── on_enter_zone       → 进入区域时
│   │   └── on_buff_apply       → buff被添加时
│   │
│   └── 周期触发 (Periodic)
│       └── every(X)            → 每X秒
│
└── 延迟触发
    └── after(X)                → X秒后
```

**示例**：
```
技能A：生命值低于30%时自动触发
技能B：受到伤害时触发
技能C：每5秒触发一次
技能D：施放后3秒触发
```

### WHAT - 效果/行为

```
元语素：效果

效果分类
├── 即时效果 (Instant)
│   ├── damage              → 伤害
│   ├── heal               → 治疗
│   ├── shield             → 护盾
│   ├── restore_mp         → 恢复能量
│   └── modify_resource    → 修改资源
│
├── 持续效果 (Continuous)
│   ├── apply_buff         → 添加buff/debuff
│   ├── remove_buff        → 移除buff
│   ├── damage_over_time   → 持续伤害 (DoT)
│   ├── heal_over_time     → 持续治疗 (HoT)
│   └── drain              → 吸血/吸蓝
│
├── 移动效果 (Movement)
│   ├── dash               → 冲刺
│   ├── teleport           → 传送
│   ├── pull               → 拉取
│   ├── knockback          → 击退
│   └── leap               → 跳跃
│
├── 状态效果 (State)
│   ├── stun               → 眩晕
│   ├── silence            → 沉默
│   ├── root               → 定身
│   ├── fear               → 恐惧
│   ├── invisibility       → 隐身
│   └── invulnerable       → 无敌
│
└── 特殊效果 (Special)
    ├── summon             → 召唤
    ├── polymorph          → 变形
    ├── resurrect          → 复活
    ├── crowd_control      → 控制时间累积
    └── immunity           → 免疫
```

**效果属性**：
```
效果元语素
├── 数值 (Magnitude)
│   ├── fixed(X)          → 固定值
│   ├── percent(X)        → 百分比
│   ├── scaling(Y, X)     → 随Y属性X%变化
│   └── curve(level)      → 随等级曲线
│
├── 标签 (Tags)
│   ├── buff_type         → buff类型标签
│   ├── can_be_purged    → 是否可驱散
│   └── priority          → 优先级
│
└── 持续时间 (Duration)
    ├── instant           → 立即
    ├── timed(X)         → 持续X秒
    └── infinite         → 永久
```

### HOW - 执行方式

```
元语素：执行方式

执行模式
├── 即时执行 (Immediate)
│   └── 效果立即应用
│
├── 延迟执行 (Delayed)
│   └── after(X)        → X秒后执行
│
├── 引导执行 (Channel)
│   ├── duration(X)     → 引导X秒
│   ├── interruptible    → 是否可中断
│   └── tick_every(Y)   → 每Y秒触发一次
│
├── 分段执行 (Phased)
│   ├── phase_1         → 第一阶段
│   ├── phase_2         → 第二阶段
│   └── phase_N         → 第N阶段
│
├── 周期执行 (Periodic)
│   ├── every(X)        → 每X秒
│   ├── times(N)        → 共N次
│   └── until(condition) → 直到条件满足
│
└── 条件执行 (Conditional)
    ├── when(condition) → 条件满足时
    ├── stacking(X)      → 叠加X层时触发
    └── stacks_expire   → 层数消失时触发
```

## 三、组合模式

### 模式 1：简单直接释放

```
描述：施放 → 立即产生效果

WHEN: 玩家施放
WHO:  target (当前目标)
WHAT: damage(100)
HOW:  immediate
```

### 模式 2：引导+周期性效果

```
描述：引导 → 周期产生效果 → 结束

WHEN: 玩家施放
WHO:  target
WHAT: damage(20)
HOW:  channel(duration=3s, tick_every=1s)
```

### 模式 3：延迟触发

```
描述：施放 → 延迟X秒 → 产生效果

WHEN: 玩家施放
WHO:  target
WHAT: damage(100)
HOW:  delayed(after=2s)
```

### 模式 4：被动触发

```
描述：条件满足 → 自动产生效果

WHEN: health_below(30%)
WHO:  self
WHAT: heal(50)
HOW:  immediate, passive
```

### 模式 5：多段技能

```
描述：施放 → 准备(延迟) → 生效 → 冷却

WHEN: 玩家施放
WHO:  target
WHAT: [phase_1: damage(50), phase_2: damage(100)]
HOW:  phased([
    phase_1: delay(0.5s),
    phase_2: immediate,
    cooldown: 3s
])
```

### 模式 6：光环效果

```
描述：存在时 → 持续对范围内单位生效

WHEN: always (光环激活)
WHO:  allies_in_range(10m, except_self)
WHAT: buff(attack_speed_up_20%)
HOW:  continuous(infinite)
```

### 模式 7：叠加触发

```
描述：叠加N层 → 触发效果 → 重置层数

WHEN: stacks(5)
WHO:  target
WHAT: stun(2s)
HOW:  on_stack_max
```

### 模式 8：Buff/Debuff

```
描述：添加 → 持续存在 → 产生周期效果 → 移除

WHEN: 施放
WHO:  target
WHAT: [
    buff_type: slow,
    duration: 5s,
    effects: [
        modifier(move_speed, -30%),
        damage_over_time(10, every=1s)
    ]
]
HOW:  continuous(duration=5s)
```

## 四、问题拆解

### 问题 1：WHEN 触发条件太多怎么办？

```
分解：
├── 触发时机 (Trigger)
│   └── 主动 / 被动
│
├── 触发频率 (Frequency)
│   └── 一次 / 每次满足 / 冷却内一次
│
└── 触发验证 (Validation)
    └── 条件A AND 条件B AND NOT 条件C
```

**解决方案**：
- 用**触发器注册表**管理不同触发条件
- 用**频率限制**防止过度触发
- 用**条件表达式**组合复杂条件

### 问题 2：WHAT 效果太复杂怎么办？

```
分解：
├── 效果类型 (Type)
│   └── 伤害 / 治疗 / 状态 / ...
│
├── 效果数值 (Magnitude)
│   └── 基础值 + 修正值
│
├── 效果目标 (Target)
│   └── 单一 / 多个 / 范围
│
└── 效果条件 (Condition)
    └── 无条件 / 条件通过生效
```

**解决方案**：
- 用**效果注册表**管理不同效果类型
- 用**数值计算管线**处理复杂数值
- 用**目标选择器**处理多目标

### 问题 3：HOW 执行流程太多怎么办？

```
分解：
├── 调度模式 (Schedule)
│   └── 立即 / 延迟 / 周期 / 引导
│
├── 生命周期 (Lifecycle)
│   └── 开始 → 执行中 → 结束
│
└── 中断处理 (Interruption)
    └── 可中断 / 不可中断 / 中断处理
```

**解决方案**：
- 用**阶段管线**组织复杂流程
- 用**状态机**管理生命周期
- 用**事件系统**处理中断

### 问题 4：WHO 目标选择太复杂怎么办？

```
分解：
├── 选择器链 (Selector Chain)
│   └── 基础选择 → 过滤 → 排序 → 限制数量
│
└── 范围计算 (Range Calculation)
    └── 点 / 圆 / 扇形 / 直线
```

**解决方案**：
- 用**管道式选择器**组合选择条件
- 用**空间索引**加速范围查询

## 五、抽象 vs 实现对照

| 本质抽象 | 技术实现 |
|---------|---------|
| **WHO** - 目标选择 | GameplayTags, Selector, Range |
| **WHEN** - 触发时机 | Trigger, Event, Condition |
| **WHAT** - 效果/行为 | Modifier, Action, Effect |
| **HOW** - 执行方式 | Pipeline, Phase, Scheduler |

```
本质需求                     技术实现
─────────────────────────────────────────────
"对谁"    ──────────────→  目标系统 (TargetSystem)
"什么时候" ───────────→  触发系统 (TriggerSystem)
"做什么"  ──────────────→  效果系统 (EffectSystem)
"怎么做"  ──────────────→  执行系统 (ExecutionSystem)
─────────────────────────────────────────────
     ↑                       ↑
  本质抽象              实现细节
```

## 六、设计检查清单

设计一个技能时，先回答这四个问题：

```
□ WHO - 目标是什么？
  - 单一目标还是多目标？
  - 需要过滤条件吗？
  - 有范围限制吗？

□ WHEN - 什么时候触发？
  - 主动施放还是被动触发？
  - 有触发条件吗？
  - 有触发冷却吗？

□ WHAT - 产生什么效果？
  - 即时效果还是持续效果？
  - 效果可以叠加吗？
  - 有副作用吗？

□ HOW - 怎么执行？
  - 立即执行还是延迟？
  - 可以被打断吗？
  - 有多阶段吗？
```
