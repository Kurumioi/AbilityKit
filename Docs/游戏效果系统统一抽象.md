# 游戏效果系统 - 统一抽象

> **核心理念**：Buff、主动技能、被动技能、引导技能、光环效果...本质上都是同一种东西，只是配置参数不同。

---

## 一、打破壁垒：一切皆为「效果」

### 传统思维的局限

```
传统设计：
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│  Buff   │  │ 技能    │  │ 被动    │  │ 光环    │
└────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘
     │             │             │             │
     ▼             ▼             ▼             ▼
  独立实现     独立实现     独立实现     独立实现
```

### 统一抽象的思维

```
统一设计：
┌─────────────────────────────────────────────────────────┐
│                    Continuous Effect                     │
│                     (持续效果)                           │
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 配置参数                                          │    │
│  │  - WHO: 目标是谁？                              │    │
│  │  - WHEN: 什么时候触发？                         │    │
│  │  - WHAT: 做什么效果？                           │    │
│  │  - HOW: 怎么执行？                              │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
          │              │              │              │
          ▼              ▼              ▼              ▼
      ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
      │  Buff   │  │ 技能    │  │ 被动    │  │ 光环    │
      └─────────┘  └─────────┘  └─────────┘  └─────────┘
```

## 二、统一配置模型

### 2.1 通用效果配置

```csharp
public interface IEffectConfig
{
    // ===== WHO - 目标 ===== //
    TargetSelector Target { get; }           // 目标选择器
    RangeShape Range { get; }               // 范围形状
    float RangeValue { get; }               // 范围大小
    
    // ===== WHEN - 时机 ===== //
    TriggerMode TriggerMode { get; }         // 触发模式
    TriggerCondition TriggerCondition { get; } // 触发条件
    float Cooldown { get; }                 // 触发冷却
    
    // ===== WHAT - 效果 ===== //
    IReadOnlyList<Effect> Effects { get; }  // 效果列表
    GameplayTagContainer Tags { get; }       // 标签
    ModifierData[] Modifiers { get; }      // 属性修改
    
    // ===== HOW - 执行方式 ===== //
    ExecutionMode ExecutionMode { get; }     // 执行模式
    float Duration { get; }                 // 持续时间
    float Interval { get; }                 // 执行间隔
    bool Interruptible { get; }              // 是否可中断
}
```

### 2.2 配置参数枚举

```csharp
// WHO - 触发模式
public enum TriggerMode
{
    Active,          // 主动施放
    Passive,         // 被动触发
    Aura,            // 光环激活
    Toggle           // 开关切换
}

// WHEN - 触发条件
public enum TriggerCondition
{
    None,           // 无条件（主动施放）
    OnDamaged,      // 受到伤害时
    OnKill,         // 击杀时
    OnDeath,        // 死亡时
    OnHit,          // 命中时
    OnCrit,         // 暴击时
    HealthBelow,     // 生命值低于
    HealthAbove,     // 生命值高于
    BuffApplied,     // Buff被添加时
    BuffExpired,     // Buff消失时
    Periodic         // 周期触发
}

// HOW - 执行模式
public enum ExecutionMode
{
    Immediate,      // 立即执行一次
    Delayed,        // 延迟执行
    Continuous,     // 持续执行
    Phased,         // 分阶段执行
    Channeled       // 引导执行
}
```

## 三、统一配置示例

### 3.1 Buff = 被动 + 持续

```
配置：
─────────────────────────────────────
TriggerMode:  Passive
TriggerCondition:  OnApply (施放时添加)
ExecutionMode:  Continuous
Duration:  5s
Interval:  1s
Effects:
  - damage_over_time(10)  // 每秒造成10点伤害
Tags:
  - Debuff
  - Poison
  - Purgable
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Passive, TriggerCondition.OnApply)
    .WithExecution(ExecutionMode.Continuous, duration: 5f, interval: 1f)
    .WithEffect(damage => target.TakeDamage(10))
    .WithTags("Debuff", "Poison", "Purgable");
```

### 3.2 主动技能 = 主动 + 立即

```
配置：
─────────────────────────────────────
TriggerMode:  Active
TriggerCondition:  None (无条件触发)
ExecutionMode:  Immediate
Effects:
  - damage(100)
  - apply_buff(Stun, duration: 2s)
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Active)
    .WithExecution(ExecutionMode.Immediate)
    .WithEffects(
        new DamageEffect(100),
        new ApplyBuffEffect("Stun", duration: 2f)
    );
```

### 3.3 被动技能 = 被动 + 条件触发 + 立即

```
配置：
─────────────────────────────────────
TriggerMode:  Passive
TriggerCondition:  OnDamaged (受到伤害时)
TriggerCondition:  Probability(20%) (20%概率)
ExecutionMode:  Immediate
Effects:
  - damage_to_attacker(50)  // 反击伤害
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Passive, TriggerCondition.OnDamaged)
    .WithCondition(ctx => ctx.Random.NextFloat() < 0.2f)
    .WithExecution(ExecutionMode.Immediate)
    .WithEffect(ctx => ctx.Caster.DealDamageTo(ctx.Attacker, 50));
```

### 3.4 光环 = 被动 + 持续 + 范围

```
配置：
─────────────────────────────────────
TriggerMode:  Passive
TriggerCondition:  Aura (光环激活)
ExecutionMode:  Continuous
Duration:  Infinite (永久)
Interval:  1s (每秒检查)
Target:  AlliesInRange
Range:  Circle(10m)
Effects:
  - modify_attribute(AttackSpeed, +20%)
  - apply_tag(SpeedBoosted)
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Passive, TriggerCondition.Aura)
    .WithExecution(ExecutionMode.Continuous, duration: float.MaxValue, interval: 1f)
    .WithTarget(TargetSelector.AlliesInRange, range: 10f)
    .WithModifier(ModifierData.PercentAdd("AttackSpeed", 0.2f))
    .WithEffect(ctx => ctx.Targets.ForEach(t => t.AddTag("SpeedBoosted")));
```

### 3.5 引导技能 = 主动 + 引导

```
配置：
─────────────────────────────────────
TriggerMode:  Active
ExecutionMode:  Channeled
Duration:  3s
Interval:  1s (每秒触发)
Effects:
  - damage(50)  // 每秒造成50点伤害
Interruptible:  true
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Active)
    .WithExecution(ExecutionMode.Channeled, duration: 3f, interval: 1f)
    .WithEffect(ctx => ctx.Target.TakeDamage(50))
    .WithInterruptible(true);
```

### 3.6 叠加触发 = 被动 + 堆叠 + 条件

```
配置：
─────────────────────────────────────
TriggerMode:  Passive
TriggerCondition:  OnHit (每次命中时)
ExecutionMode:  Continuous
Stackable:  true
MaxStacks:  5
StackEffect:
  - slow(-10% per stack)  // 每层减速10%
TriggerOnMaxStacks:
  - stun(2s)
  - reset_stacks
```

```csharp
// 代码实现
var config = EffectConfig.Create()
    .WithTrigger(TriggerMode.Passive, TriggerCondition.OnHit)
    .WithExecution(ExecutionMode.Continuous)
    .WithStackable(maxStacks: 5)
    .WithStackEffect(ctx => ctx.Target.ModifyAttribute("MoveSpeed", -0.1f * ctx.Stacks))
    .WithOnMaxStacks(ctx =>
    {
        ctx.Target.ApplyStun(2f);
        ctx.ResetStacks();
    });
```

## 四、统一管理机制

### 4.1 效果注册表

```csharp
public class EffectRegistry
{
    // 所有效果统一注册
    public ContinuousHandle Start(IEffectConfig config, EffectContext ctx)
    {
        // 统一处理
        var instance = CreateInstance(config, ctx);
        
        // 处理触发时机
        switch (config.TriggerMode)
        {
            case TriggerMode.Active:
                // 立即执行并注册
                return StartActiveEffect(instance);
                
            case TriggerMode.Passive:
                // 注册被动触发器
                return RegisterPassiveEffect(instance);
                
            case TriggerMode.Aura:
                // 注册光环效果
                return RegisterAuraEffect(instance);
        }
    }
    
    // 统一的中断/完成逻辑
    public void End(ContinuousHandle handle, EndReason reason)
    {
        var instance = GetInstance(handle);
        
        // 统一清理
        instance.RemoveModifiers();     // 移除属性修改
        instance.RemoveTags();          // 移除标签
        instance.CancelScheduled();      // 取消定时任务
        instance.NotifyEnd(reason);     // 通知结束事件
    }
}
```

### 4.2 效果实例

```csharp
public class EffectInstance
{
    public IEffectConfig Config { get; }
    public EffectContext Context { get; }
    public ContinuousState State { get; set; }
    public int Stacks { get; set; }
    public float ElapsedTime { get; set; }
    
    // ===== 统一的效果执行 ===== //
    public void Execute()
    {
        // 1. 检查执行条件
        if (!CanExecute()) return;
        
        // 2. 执行效果
        foreach (var effect in Config.Effects)
        {
            effect.Apply(this);
        }
        
        // 3. 更新状态
        ExecuteCount++;
    }
    
    // ===== 统一的生命周期 ===== //
    public void Start()
    {
        // 应用修改器
        foreach (var mod in Config.Modifiers)
        {
            Context.Target.AddModifier(mod);
        }
        
        // 添加标签
        foreach (var tag in Config.Tags)
        {
            Context.Target.AddTag(tag);
        }
        
        // 启动调度
        if (Config.ExecutionMode == ExecutionMode.Continuous)
        {
            SchedulePeriodic();
        }
        
        State = ContinuousState.Active;
    }
    
    public void End(EndReason reason)
    {
        // 移除修改器
        Context.Target.RemoveModifiersBySource(Config.SourceId);
        
        // 移除标签
        foreach (var tag in Config.Tags)
        {
            Context.Target.RemoveTag(tag);
        }
        
        State = ContinuousState.Ended;
    }
}
```

## 五、配置对比表

| 维度 | Buff | 主动技能 | 被动技能 | 光环 |
|------|------|---------|---------|------|
| **TriggerMode** | Passive | Active | Passive | Passive |
| **TriggerCondition** | OnApply | None | 各种条件 | Aura |
| **ExecutionMode** | Continuous | Immediate | Conditional | Continuous |
| **Duration** | 有限(5s, 10s) | 0 | 条件决定 | Infinite |
| **Interruptible** | 通常可 | 视技能而定 | 否 | 否 |

**结论**：它们使用**完全相同的配置结构**，只是参数值不同。

## 六、统一的好处

### 6.1 代码复用

```
传统做法：
├── BuffSystem.StartBuff()
├── SkillSystem.CastSkill()
├── PassiveSystem.RegisterPassive()
├── AuraSystem.StartAura()
└── 每个系统都有自己的生命周期管理

统一做法：
├── EffectRegistry.Start(config)
└── 所有效果使用同一套生命周期管理
```

### 6.2 统一的查询

```csharp
// 查询所有激活中的效果（不管是什么类型）
var allEffects = registry.GetActiveEffects();

// 查询所有"不可驱散"的效果
var unpurgable = registry.GetByTag("Unpurgable");

// 中断所有"眩晕"效果
registry.InterruptByTag("Stun");

// 查询某个目标身上的所有效果
var targetEffects = registry.GetByTarget(targetId);
```

### 6.3 统一的持久化

```csharp
// 所有效果使用统一的序列化格式
[Serializable]
public class EffectSaveData
{
    public string ConfigId;      // 配置ID
    public int SourceId;         // 来源
    public float RemainingTime;  // 剩余时间
    public int Stacks;          // 层数
    // ...
}
```

### 6.4 统一的调试/日志

```csharp
// 所有效果使用统一的事件日志
EventLog.Record("EffectStarted", config.Name, target.Name, trigger);
EventLog.Record("EffectTick", config.Name, tickCount);
EventLog.Record("EffectEnded", config.Name, reason);
```

## 七、设计原则

### 原则1：配置驱动，非代码驱动

```
❌ 错误：为每种效果写专门的代码

class PoisonDebuff : ISkillEffect { ... }
class StunDebuff : ISkillEffect { ... }
class SlowDebuff : ISkillEffect { ... }

✅ 正确：用配置描述效果，代码统一处理

var config = EffectConfig.Create()
    .WithEffect(damage_over_time(10))
    .WithTags("Poison", "Purgable")
    .WithDuration(5f);

registry.Start(config);
```

### 原则2：参数化，非特殊化

```
❌ 错误：用特殊类型区分不同效果

class Buff { }
class Skill { }
class Passive { }
class Aura { }

✅ 正确：用参数区分，都是效果的不同配置

IEffectConfig config = ...;  // 可以是任何效果
registry.Start(config);
```

### 原则3：接口组合，非继承

```
❌ 错误：继承树

          IEffect
          /    \
      Buff    Skill
        \      /
      Poison  Damage

✅ 正确：接口组合

IEffectConfig config = EffectConfig.Create()
    .WithTrigger(...)     // ITagRequirementsSource
    .WithModifier(...)    // IModifierSource
    .WithEffects(...)     // IEffectSource
    .WithTags(...)        // ITagSource
    // 所有特性都是可选的接口
```
