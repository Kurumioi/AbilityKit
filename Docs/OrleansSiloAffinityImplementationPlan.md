# Orleans 专用部署与亲和性落地计划

## 优先级
1. **部署角色与亲和性配置骨架**：先把“哪些 silo 承载哪些逻辑域”显式化。
2. **配置绑定与验证**：把角色/分组/亲和性从代码常量迁到配置。
3. **启动入口接入**：Gateway/Host/Smoke 启动时读取新配置。
4. **grain 路由预留点**：为房间/战斗 grain 预留按组分配的接入点。
5. **观测与诊断**：在 health / diagnostics 中暴露当前角色与分组信息。

## 当前实施顺序
- 先新增部署模型与配置 section 常量
- 再接入 `IConfiguration` 绑定与校验
- 然后在启动入口使用这些配置
- 最后再逐步给房间/战斗 grain 加路由元数据

## 预期收益
- 减少跨进程调用
- 支持专用 silo 部署
- 让高频协作逻辑尽量同组落点
- 让后续扩容与隔离更可控
